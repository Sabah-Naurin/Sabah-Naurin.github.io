Introduction to Java

Java programs might seem scary at first, especially if you’re used to Python, where you can do a lot with just one or two lines. But once you get the hang of Java’s structure, it’s pretty simple! Think of it like a recipe where every step has to be written clearly.

The Structure of a Java Program

Every Java program starts with a class. The file name should match the class name (case-sensitive) and have the .java extension.

How a Basic Java Program Looks

Here’s the simplest Java program that prints "Hello, World!" to the screen (And corresponding Python code):


public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
                      

print("Hello, World!")
                      
Hello, World!

Let’s Break It Down

Public class Main { ... } Declares a class named Main. The file must be saved as Main.java.
Every Java program starts with a class. The class name must match the filename (e.g., Main.java).
public static void main(String[] args) This is the main method, the entry point of your program. Java starts executing here. Each keyword has a specific purpose:
  • public: Accessible by other parts of the program.
  • static: You don’t need an object to run this method.
  • void: This method doesn’t return a value.
{ ... } Curly braces group blocks of code. Every class and method must have them.
System.out.println("Hello, World!"); Prints the text "Hello, World!" to the console and moves to the next line.
Don’t forget the semicolon!

Java Programs Are Always in a Class

In Java, everything must be inside a class. This is one of the biggest differences between Java and Python. Even for the simplest program, you must define a class to organize your code.

In Java, you can’t write code outside a class. In Python, you don’t need to use a class unless your program requires object-oriented features.

Example: Adding Two Numbers


public class Main {
  public static void main(String[] args) {
      int a = 5;
      int b = 10;
      System.out.println("Sum: " + (a + b));
  }
}
                    

a = 5
b = 10
print("Sum:", a + b)
                    
Sum: 15

Why Does Java Require Classes?

Java is designed to be a pure object-oriented language.

  • Classes help organize and structure code.
  • They make Java programs scalable and maintainable, especially for large projects.
  • While it may seem like extra work for small programs, this structure becomes very useful in complex applications.

File Name and Class Name Match

In Java, the file name must match the public class name exactly (including capitalization). This rule applies only to the class marked as public. If you violate this rule, Java will throw a compilation error.

Example:

If your class is named Main, the file must be saved as Main.java:


public class Main {  // The public class is named Main
  public static void main(String[] args) {
      System.out.println("File name matches the class name.");
  }
}
What Happens if the File Name Doesn’t Match?

If you save the above code as Example.java, Java will throw an error like:

Error: class Main is public, should be declared in a file named Main.java

Case Sensitivity

Java is a case-sensitive language, which means that identifiers (like variable names, method names, and class names) must match exactly in terms of uppercase and lowercase letters. Even a small difference in capitalization will lead to an error.

Example:


public class Main {
  public static void main(String[] args) {
      int Number = 10;  // an interger variable 'Number' with uppercase N and a value of 10
      int number = 20;  // an interger variable 'number' with lowercase n and a value of 20

      System.out.println(Number);  // Outputs: 10
      System.out.println(number);  // Outputs: 20
  }
}
Common Mistake:

System.out.println("Hello"); // Correct
system.out.println("Hello"); // Incorrect: "system" must be "System"

Curly Braces ({})

Curly braces are used to define blocks of code in Java. Every class, method, and control statement (like loops and if statements) must be enclosed in curly braces. This ensures that Java knows where the block of code starts and ends.

Example:


public class Main {  // Open curly brace for class
  public static void main(String[] args) {  // Open curly brace for main method
      System.out.println("Hello, World!"); // Statement inside method
  }  // Close curly brace for main method
}  // Close curly brace for class

Semicolons (;)

In Java, every statement must end with a semicolon (;). This rule applies to executable statements, like method calls, variable declarations, and return statements.

Example:


public class SemicolonExample {
  public static void main(String[] args) {
      int number = 10;  // Correct: Ends with a semicolon
      System.out.println("The number is " + number);  // Correct

      // Uncommenting the next line will cause a syntax error because the semicolon is missing
      // System.out.println("This will cause an error")
  }
}
Forgetting a semicolon is a very common mistake for beginners.

What Does System.out.println("") Do?

System.out.println("") is Java’s way of printing messages to the console.

  • System: A built-in Java class that provides tools.
  • out: A tool inside System used to send output.
  • println: Stands for print line, which prints the message and moves to the next line.

If you use it without anything inside (System.out.println("");), it simply prints an empty line.
Similar to Python's print() statement. It’s how your program "talks" to you!
